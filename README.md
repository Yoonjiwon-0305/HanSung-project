# 🍊 Vision Mandarin: 딥러닝을 활용한 귤 선별 시스템

[cite_start]**Vision Mandarin**은 귤의 외관 정보를 기반으로 딥러닝을 활용하여 자동으로 귤을 분류하는 스마트 선별 시스템입니다[cite: 19]. [cite_start]농산물 유통 과정에서의 품질 관리 효율성 향상을 목표로 라즈베리파이와 YOLO 모델을 결합하여 설계되었습니다[cite: 20].

## 📌 프로젝트 개요
* [cite_start]**목표**: 이미지 처리와 딥러닝 모델을 활용한 귤의 불량 및 크기 자동 분류 시스템 구현[cite: 20].
* [cite_start]**핵심 기능**: 카메라 스캔을 통해 '대, 중, 소, 불량' 4가지 클래스로 실시간 자동 분류[cite: 21].
* [cite_start]**기대 효과**: 저비용·고효율 솔루션으로 소규모 농가나 스마트팜 시스템에 적용 가능한 실용적 기반 제공[cite: 23].

## 🛠 시스템 구성 (System Architecture)
[cite_start]본 시스템은 하드웨어 제어부와 AI 추론부로 나누어 두 대의 라즈베리파이를 TCP/IP 통신으로 연동하여 작동합니다[cite: 40, 84].

### [cite_start]1. 주요 하드웨어 모듈 [cite: 52, 53]
| 부품명 | 역할 및 동작 원리 |
| :--- | :--- |
| **Raspberry Pi 4** | 웹캠 이미지 수신 및 Edge TPU를 통한 AI 모델 추론 수행. |
| **Edge TPU (Coral)** | 라즈베리파이의 추론 속도를 실시간 수준으로 향상시키는 가속기. |
| **USB 웹캠 (4대)** | 스캔 구역에서 귤 이미지를 다각도로 촬영하여 추론 결과 전달. |
| **기어드 모터** | 컨베이어 벨트를 회전시켜 귤을 스캔 구역으로 이동. |
| **서보모터** | 분류 신호에 따라 회전 각도를 조절하여 귤을 해당 구역으로 투하. |
| **초음파 센서** | 귤의 도달을 감지하여 웹캠 촬영 및 분류 동작의 트리거 역할 수행. |

### 2. 소프트웨어 및 AI 파이프라인
* [cite_start]**Model**: YOLO 11[cite: 34].
* [cite_start]**Dataset**: Roboflow를 통해 증강된 약 3,800장의 커스텀 데이터셋 활용[cite: 75].
* [cite_start]**Optimization**: Google Colab에서 학습 후 Edge TPU 사용을 위해 `edgetpu.tflite` 형식으로 양자화 및 변환[cite: 77, 82].
* [cite_start]**Environment**: Python, OpenCV, TensorFlow Lite[cite: 40, 79].

## [cite_start]🚀 동작 프로세스 (Workflow) [cite: 61, 62, 64, 68, 70, 71]
1. **감지**: 컨베이어 벨트 위의 귤을 초음파 센서가 감지하면 벨트가 정지합니다.
2. **명령**: 분류기 라즈베리파이가 추론 라즈베리파이에게 추론 시작 명령을 보냅니다.
3. **추론**: 4대의 카메라가 3초 동안 귤을 탐지하며, 정확도를 높이기 위해 가장 많이 카운트된 결과를 선정합니다.
4. **분류**: 추론 결과(`defective`, `Large`, `Medium`, `Small`)에 따라 서보모터가 차단막을 오픈합니다.
5. **배출**: 스캔 구역 바닥면이 회전하여 귤이 해당 칸으로 미끄러져 내려갑니다.
6. **재시작**: 차단막이 닫히면 컨베이어 벨트가 다시 작동하여 다음 과정을 반복합니다.

## 📈 문제 해결 및 성과 (Troubleshooting)
* [cite_start]**HW 안정화**: 기어드 모터 교체 및 브라켓 설계를 통해 축 고정 문제를 해결하고 구동 메커니즘을 안정화했습니다[cite: 587, 594].
* [cite_start]**구조 개선**: 귤이 튕겨 나가는 문제를 해결하기 위해 경사각을 60도에서 완만한 각도로 재설계하여 분류 정확도를 높였습니다[cite: 597, 600].
* [cite_start]**SW 최적화**: 가상환경을 구축하여 패키지 버전을 정밀하게 맞춤으로써 Edge TPU용 모델 변환 오류를 극복했습니다[cite: 606].
